## 🧠 How I Think (Core Principles)

### 01. Understand Before Building
*I don't start with code. I start with the problem.*
Before writing a line of React or setting up a database schema, I analyze constraints, business metrics, and user flows. Building the wrong feature perfectly is still a failure.

### 02. Design For Humans
*Users shouldn't have to think.*
Good interfaces disappear. Every interaction, page transition, validation error, and click state should feel predictable, fast, and satisfying.

### 03. Performance Is A Feature
*Fast products win.*
Performance directly impacts trust, engagement, retention, and SEO rankings. A gorgeous application that feels slow is a broken application.

### 04. Build For Growth
*The first version is never the last version.*
I build structured, self-documenting codebases that scale cleanly. My architectures are decoupled, type-safe, and simple to navigate.
